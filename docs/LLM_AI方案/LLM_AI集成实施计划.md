# 基于 LLM 的斗地主 AI 集成方案

本方案旨在通过大语言模型（LLM）替换或增强现有的基于规则的 AI，使 AI 能够通过“推理”游戏局势做出更人性化、更具战略性的决策。

## 1. 核心变更建议

### 1.1 核心逻辑集成
#### [修改] `doudizhu.js`
- **实现 `getLLMDecision(context)`**：一个异步函数，负责将当前过去出牌记录、当前游戏状态（手牌、上家出牌、倍数、剩余牌数等）封装成 Prompt，并调用 LLM API。
- **更新 `aiBid(playerId)`**：支持 `async/await`，调用 LLM 决定叫分（0-3分）。
- **更新 `doAIPlay(playerId)`**：调用 LLM 选择出牌组合，而非仅依靠贪心算法。
- **安全兜底机制**：如果 LLM 返回了非法牌型（例如出了手中没有的牌）或接口超时，系统将自动回退到现有的规则算法（`findBestPlay`），确保游戏流程不中断。

### 1.2 用户界面优化
#### [修改] `doudizhu.html`
- **增加“智能 AI”开关**：允许用户在“普通”（也就是规则出牌）和“大师”（LLM出牌）之间切换。
- **AI 动态显示**：在 LLM 思考时，头像上方显示“思考中...”等更高级的状态提示。

## 2. 提示词工程 (Prompt Engineering)

系统将按如下逻辑引导 LLM：

### 2.1 叫分阶段 Prompt
- **Context**: 你的手牌是 [X, Y, Z...]，底牌未知，当前最大叫分是 X。
- **Task**: 评估你的手牌强度，给出你的叫分建议。
- **Format**: 仅输出数字 0, 1, 2 或 3。

### 2.2 出牌阶段 Prompt
- **Past Rounds**: 过去几轮的出牌情况，结构应该是一个列表，每个元素是一轮出牌情况{玩家：牌}
- **Context**: 你的手牌是 [...]，当前身份是 [地主/农民]，上家出的牌是 [...]，场上已出的关键牌有 [...]。
- **Task**: 根据斗地主高级策略，决定是压牌还是过牌。
- **Format**: 返回 JSON 格式，如 `{"action": "play", "cards": ["JOKER_BIG", "2", "2"]}` 或 `{"action": "pass"}`。

## 3. 验证计划

1. **逻辑闭环测试**：通过日志检查 LLM 返回的 JSON 是否能正确解析并转化为游戏动作。
2. **对局质量观察**：对比常规 AI，观察 LLM 是否学会了“拆牌”、“留炸”、“配合队友”等高级战术。
3. **性能损耗评估**：测试 API 请求延迟对游戏流畅度的影响，并据此调整 UI 提示时长。
